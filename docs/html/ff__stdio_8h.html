Descrição detalhada

FreeRTOS+FAT-like stdio wrapper over ChaN FatFs for RP2040 projects.

This header declares a thin compatibility layer that mirrors a subset of the FreeRTOS+FAT API (ff_* namespace) using ChaN's FatFs under the hood. It provides familiar functions such as ff_fopen, ff_fread, ff_fwrite, directory traversal (ff_findfirst/ff_findnext) and others, mapping them to the corresponding FatFs calls. This facilitates porting code and running filesystem examples/tests on the Raspberry Pi Pico (RP2040).

Typical usage:

  • Mount the drive with FatFs (f_mount) or use helper code from examples
  • Use ff_fopen/ff_fread/ff_fwrite etc. as you would on FreeRTOS+FAT
  • Close files with ff_fclose and unmount when finished

The types FF_FILE and FF_Stat_t wrap FatFs types (FIL, FILINFO).

Nota
This is a compatibility shim; it does not attempt to be a full reimplementation of FreeRTOS+FAT. Consult the implementation for supported calls and semantics.
