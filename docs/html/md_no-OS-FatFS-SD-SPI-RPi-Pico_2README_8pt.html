no-OS-FatFS-SD-SPI-RPi-Pico (Português)

Este projeto integra o FatFs (biblioteca de sistema de arquivos FAT) com um driver de cartão SD via SPI para o Raspberry Pi Pico (RP2040). Além da biblioteca, há projetos de exemplo prontos para compilar, um CLI simples e testes que podem ser usados como base para aplicações reais (ex.: data logger).

  • Suporte a múltiplos cartões SD
  • Suporte a múltiplas instâncias de SPI
  • Suporte a RTC para timestamps
  • API de alto nível do FatFs e camada compatível com FreeRTOS+FAT (ff_stdio.h)

Pré‑requisitos

  • Raspberry Pi Pico
  • Placa/suporte para microSD com interface SPI
  • SDK C/C++ do Raspberry Pi Pico configurado
  • CMake e toolchain do Pico

Como compilar

Exemplo genérico (substitua pelo exemplo desejado):

# Exemplo CLI completo
cd example
mkdir -p build && cd build
cmake ..
make -j

Outros exemplos:

# Exemplo simples
cd simple_example && mkdir -p build && cd build && cmake .. && make -j
# Exemplo de configuração dinâmica
cd dynamic_config_example && mkdir -p build && cd build && cmake .. && make -j

O binário UF2 será gerado em build/. Entre em modo BOOTSEL no Pico, monte como unidade USB e arraste o .uf2 correspondente.

Como usar (visão geral)

  • Conecte um terminal serial ao Pico (USB). Ferramentas úteis: tio, minicom, PuTTY.
  • Exemplos:
    • simple_example: monta o volume, escreve uma linha em filename.txt, desmonta.
    • example (CLI): comandos como format, mount, ls, cat, mkdir, cd, start_logger. Digite help para ver a ajuda.
    • dynamic_config_example: demonstra a criação dos objetos spi_t e sd_card_t em tempo de execução (sem hw_config.c fixo).

Personalização de Hardware

A configuração de pinos e parâmetros é feita via objetos:

  • spi_t (pinos MISO/MOSI/SCK, baud_rate, drive strength)
  • sd_card_t (pino CS, Card Detect, nome lógico, etc.)

Você pode optar por configurar isso estaticamente (em hw_config.c) ou dinamicamente (ver dynamic_config_example/hw_config.cpp). Consulte o README.md original para detalhes e tabelas de ligação de pinos.

Geração de Documentação (Doxygen)

Este repositório contém comentários Doxygen em headers e fontes principais. Para gerar documentação local:

# Na raiz do workspace/projeto onde houver Doxyfile
# (ajuste caminhos/conteúdos conforme seu ambiente)
doxygen Doxyfile

A saída (HTML) será gerada no diretório configurado no Doxyfile (ex.: docs/html/).

Solução de Problemas (dicas rápidas)

  • Reduza o baud_rate do SPI caso esteja instável.
  • Garanta boa alimentação do cartão SD (capacitores de desacoplamento próximos ao soquete ajudam).
  • Cabos curtos e diretos; SPI opera em altas frequências.
  • Utilize Card Detect (quando disponível) para evitar corrupção ao trocar cartões com o volume montado.
