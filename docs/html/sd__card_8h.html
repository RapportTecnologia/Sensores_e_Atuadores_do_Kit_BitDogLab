Descrição detalhada

SD Card abstraction and configuration for RP2040 SPI driver.

Provides the sd_card_t structure that describes one SD card socket and aggregates runtime state and driver callbacks. The design supports:

  • Multiple SD cards per SPI controller
  • Optional Card Detect GPIO
  • Mounting state and associated FatFs volume (FATFS)

Typical flow:

  1. Define one or more spi_t instances (see sd_driver/spi.h).
  2. Define one or more sd_card_t instances referencing the desired spi_t.
  3. Call helper functions (provided by the application) to register these instances and then mount the volumes using FatFs (f_mount).

The function pointers in sd_card_t are implemented by the SPI SD driver and bound during initialization.
